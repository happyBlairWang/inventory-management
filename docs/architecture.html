Data Flow — Filter Change to Render
FilterBar.vue
User changes a filter
Dropdown emits a new value — e.g. Month: 2025-03, Warehouse: London
const { selectedPeriod, selectedLocation } = useFilters()
useFilters.js
Singleton reactive state updates
All views share one composable instance. Watchers in each view fire automatically.
watch([selectedPeriod, selectedLocation, selectedCategory], loadData)
api.js
Axios request with query params
Calls are made in parallel where possible using Promise.all.
getOrders({ warehouse: 'London', category: 'Sensors', month: '2025-03' }) // → GET /api/orders?warehouse=London&category=Sensors&month=2025-03
main.py
FastAPI receives request, filters in-memory list
Two helper functions handle all filtering. No database query — data lives in Python memory.
result = apply_filters(orders, warehouse='London', category='Sensors') result = filter_by_month(result, month='2025-03')
Pydantic
Response validated against model
FastAPI serializes with response_model=List[Order] before sending JSON.
View Component
Raw data stored in ref, derived data in computed
Vue's reactivity re-renders only what changed. Charts and tables update automatically.
const allOrders = ref([]) // raw from API const topProducts = computed(() => { // derived return allOrders.value.slice(0, 5) })

Vue Composables (Shared Logic)
useFilters()
Singleton filter state shared across all views. Any component can read or change the active filters.
selectedPeriod selectedLocation selectedCategory selectedStatus getCurrentFilters() resetFilters() hasActiveFilters
useI18n()
English / Japanese toggle. Locale persisted in localStorage. Currency auto-selects (USD / JPY).
t(key) currentLocale setLocale() currentCurrency translateProductName() translateWarehouse()
useAuth()
Mock authentication. Provides the current user profile and task list for the ProfileMenu.
currentUser isAuthenticated
